**Create Dockerfiles** ‚Äî one per app, using the multi-stage pattern from 01-RESEARCH.md Pattern 6. Each Dockerfile follows this structure:

Stage 1 `base`: `FROM node:22-alpine AS base`, `RUN corepack enable pnpm`

Stage 2 `deps`: Copy workspace root files (package.json, pnpm-workspace.yaml, pnpm-lock.yaml), all packages/ directories, and the specific app's package.json. Run `pnpm install --frozen-lockfile`.

Stage 3 `builder`: Copy full source from deps. Copy all source code. Run `pnpm turbo build --filter={app-name}`.

Stage 4 `runner`: Production stage. Set NODE_ENV=production. Create non-root user (nodejs:nextjs). Copy standalone output, static files, public directory. Expose 3000. CMD `node apps/{name}/server.js`.

For each app Dockerfile:
- `apps/api/Dockerfile` ‚Äî filter=api
- `apps/dashboard/Dockerfile` ‚Äî filter=dashboard
- `apps/web/Dockerfile` ‚Äî filter=web
- `apps/docs/Dockerfile` ‚Äî filter=docs

Important: Next.js `output: 'standalone'` is already set in each app's next.config.ts (Plan 01, per D-10).

**Create `.github/workflows/ci.yml`** ‚Äî exact pattern from 01-RESEARCH.md Pattern 7:

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile

      - uses: actions/cache@v4
        with:
          path: .turbo
          key: ${{ runner.os }}-turbo-${{ github.sha }}
          restore-keys: |
            ${{ runner.os }}-turbo-

      - name: Lint
        run: pnpm turbo lint

      - name: Type Check
        run: pnpm turbo typecheck

      - name: Build
        run: pnpm turbo build

      - name: Test
        run: pnpm turbo test
```

**Create `.github/workflows/release.yml`** ‚Äî Changesets automated release:

```yaml
name: Release
on:
  push:
    branches: [main]

concurrency: ${{ github.workflow }}-${{ github.ref }}

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
      id-token: write
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          registry-url: 'https://registry.npmjs.org'
          cache: 'pnpm'

      - run: pnpm install --frozen-lockfile

      - name: Build packages
        run: pnpm turbo build

      - name: Create Release Pull Request or Publish
        uses: changesets/action@v1
        with:
          publish: pnpm changeset publish
          title: 'chore: version packages'
          commit: 'chore: version packages'
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions secrets | NPM_TOKEN and GITHUB_TOKEN must be configured in repo settings |
| Docker images | Production containers run as non-root user |
| Docker Compose env | Production compose reads from .env file containing secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-13 | Elevation of Privilege | Dockerfile | mitigate | Production stage runs as non-root user `nextjs` (uid 1001); no root processes |
| T-01-14 | Information Disclosure | ci.yml | mitigate | Secrets accessed via `${{ secrets.* }}` ‚Äî never printed to logs; NPM_TOKEN only in release job |
| T-01-15 | Tampering | release.yml | mitigate | `--frozen-lockfile` prevents dependency manipulation; Changesets publish requires explicit version intent |
| T-01-16 | Denial of Service | docker-compose.yml | accept | Local dev only; no auth on MongoDB/Redis by design for DX; production should use Atlas or auth-enabled MongoDB |
</threat_model>
