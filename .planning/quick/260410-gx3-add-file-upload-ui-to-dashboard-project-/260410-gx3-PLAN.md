Add a working file upload UI to the dashboard's project files page. The user drops or selects files in a dropzone above the file list, files upload directly to R2 via session-authenticated internal presign routes, and then appear in the existing list via the 5s SWR poll.

Purpose: Today the dashboard file browser is read-only — the only way to populate a project is via the external `/api/v1/upload/*` SDK flow with an API key. Users expect to upload from inside the dashboard without needing to scaffold an SDK client first.

Output: A session-authed upload path (internal routes + UI) that mirrors the apps/api presign flow but swaps `withApiKey` for `auth()` + `Project.findOne({ slug, userId })`.

**Why not reuse `@uploadkitdev/react` UploadDropzone:** The SDK dropzone is designed around `UploadKitClient` which hits the **public API with an API key** via the `route` prop → `fileRouter` lookup. The dashboard uses session cookies, has no API key, and has no FileRouter. Rather than contort the SDK client, build a minimal dropzone inline — roughly 120 lines — that reuses the SDK's visual design tokens (`uk-dropzone`, `uk-file-item`, `uk-progress` classes already shipped via `@uploadkitdev/react/styles.css` which is imported in dashboard layout).

**Create `apps/dashboard/src/components/file-browser/dashboard-upload-dropzone.tsx` as a client component (`'use client'`)** with these responsibilities:

1. Render a drag-and-drop zone + hidden `<input type="file" multiple>` using the existing `uk-dropzone` class names (check `@uploadkitdev/react` styles are imported somewhere in dashboard — if not, import the CSS in the component via a side-effect import: `import '@uploadkitdev/react/styles.css';`).

2. Accept props: `{ slug: string; onUploadComplete: () => void }`.

3. Internal state: `files: Array<{ id, file, status, progress }>` where status ∈ `'idle' | 'uploading' | 'success' | 'error'`.

4. For each selected file, decide flow based on size:
   - **size ≤ 10 MiB:** POST to `/api/internal/projects/${slug}/upload/request` with `{ fileName, fileSize, contentType }`, receive `{ fileId, uploadUrl }`, then `fetch(uploadUrl, { method: 'PUT', body: file, headers: { 'Content-Type': file.type } })`, then POST to `/api/internal/projects/${slug}/upload/complete` with `{ fileId }`.
   - **size > 10 MiB:** POST to `/api/internal/projects/${slug}/upload/multipart/init` with same body, receive `{ fileId, uploadId, key, parts: [{ partNumber, uploadUrl }] }`. Slice file into 5 MiB chunks `file.slice(start, start + PART_SIZE)`, PUT each chunk to its `uploadUrl` in parallel (use `Promise.all` but cap at CONCURRENCY=3 using simple batch loop like SDK dropzone), collect ETags from each PUT response (`res.headers.get('etag')` — strip surrounding quotes), then POST to `/multipart/complete` with `{ fileId, uploadId, parts: [{ partNumber, etag }] }`. On any error, POST to `/multipart/abort` with `{ fileId, uploadId }` before surfacing the error.

5. Progress tracking: single-PUT flow can use `XMLHttpRequest` with `upload.onprogress` (fetch doesn't expose upload progress in browsers). Multipart can track progress as `(completedParts / totalParts) * 100` — good enough granularity for 10 MiB+ files. (Acceptable trade-off for a dashboard utility; do not try to aggregate per-byte progress across parts.)

6. All fetches to `/api/internal/*` use `credentials: 'same-origin'` (default) — session cookie is sent automatically. Do NOT set any `Authorization` header.

7. After ALL files in a batch finish (success or error), call `onUploadComplete()`. Individual file errors surface inline via a per-file error state (red X + tooltip with error message) — do not block other files from completing.

8. Render the dropzone with the existing visual style. Suggested structure using already-present ui primitives:
   ```tsx
   <div className="uk-dropzone" data-state={...} onDrop={...} onDragOver={...} onClick={() => inputRef.current?.click()}>
     <p><strong>Drop files here</strong> or click to browse</p>
   </div>
   <input ref={inputRef} type="file" multiple hidden onChange={...} />
   {files.length > 0 && (
     <div className="uk-file-list">{files.map(f => <FileRow key={f.id} {...f} />)}</div>
   )}
   ```

9. Accessibility: `role="button"`, `tabIndex={0}`, `aria-label="Drop files here or click to browse"`, Enter/Space key handler — same pattern as SDK dropzone (lines 381-390 of upload-dropzone.tsx).
