    **GET handler:**
    - Extract slug from `params.uploadkit[0]` (await params since Next.js 16 makes params a Promise).
    - Look up route in `config.router[slug]`.
    - If not found, return `Response.json({ error: { code: 'ROUTE_NOT_FOUND', message: 'Unknown file route' } }, { status: 404 })`.
    - Return `Response.json({ maxFileSize: parseFileSize(route.maxFileSize), maxFileCount: route.maxFileCount, allowedTypes: route.allowedTypes })`.

    **POST handler:**
    - Extract slug from params.
    - Look up route.
    - Parse JSON body: `{ fileId, file, action }`. The `action` field determines the operation:

    - Run middleware: `const metadata = await route.middleware?.({ req }) ?? {}`.
    - Call `route.onUploadComplete?.({ file: body.file, metadata })`.
    - Return `Response.json({ ok: true, metadata: result })`.
    - Wrap in try/catch — on error return `Response.json({ error: { code: 'HANDLER_ERROR', message: err.message } }, { status: 500 })`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer's Next.js app -> handler | Developer calls createUploadKitHandler with their credentials |
| Handler -> S3/R2 (BYOS) | BYOS credentials used server-side only to generate presigned URLs |
| Handler -> UploadKit API (managed) | API key proxied server-side to managed service |
| Client browser -> handler route | POST/GET from client to the developer's /api/uploadkit/[...uploadkit] route |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Information Disclosure | byos.ts | mitigate | BYOS credentials stay server-side only. `import 'server-only'` prevents accidental client bundle inclusion. S3Client created per-request or cached in module scope — never serialized to client. |
| T-04-08 | Spoofing | handler.ts POST | mitigate | Middleware function runs before onUploadComplete — developer authenticates the request in middleware. Handler does not bypass middleware. |
| T-04-09 | Tampering | handler.ts | mitigate | File size/type validated against route config before generating presigned URL. parseFileSize ensures string sizes ("4MB") are correctly converted. |
| T-04-10 | Information Disclosure | handler.ts GET | accept | GET returns route config (maxFileSize, allowedTypes) which is intentionally public — clients need it for client-side validation UX. No credentials exposed. |
| T-04-11 | Elevation of Privilege | byos.ts | mitigate | BYOS presigned URLs scoped to specific key path, content-type, and content-length. URL expires in 15 minutes. Cannot be used to overwrite arbitrary objects. |
| T-04-12 | Denial of Service | handler.ts | accept | Rate limiting is at the developer's infrastructure level (their Next.js app). SDK does not add its own rate limiting to the handler — that's the developer's responsibility or they use UploadKit managed mode which has rate limiting. |
</threat_model>
