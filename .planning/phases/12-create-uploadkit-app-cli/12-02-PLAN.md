next plan (template engine) consumes.

Purpose: cleanly separate "gather intent" from "apply intent" so template
logic (12-03+) is pure.

  <files>
    packages/create-uploadkit-app/src/prompts.ts,
    packages/create-uploadkit-app/src/index.ts,
    packages/create-uploadkit-app/src/__tests__/prompts.test.ts
  </files>
  <behavior>
    - First prompt: project name (default `my-uploadkit-app`). Validates: npm-safe name, not empty, directory doesn't already exist (unless `--force`).
    - Second prompt: template select (`next`, `sveltekit`, `remix`, `vite`) with short descriptions.
    - Third prompt: package manager (pre-selected from `detectPm()`).
    - Fourth prompt (only if template === 'vite'): TypeScript yes/no. Other templates force TS.
    - Fifth prompt: install dependencies now? (default yes unless `--no-install`).
    - Sixth prompt: git init? (default yes, skippable with `--no-git`).
    - Any flag value short-circuits the corresponding prompt.
    - `--yes` accepts all defaults and skips all prompts, requiring at minimum a project name positional argument.
    - Ctrl+C at any prompt calls `isCancel` path → exit 130 with a "Cancelled" message via `picocolors.yellow`.
  </behavior>
  <action>
    Implement `runPrompts(parsed: ParsedFlags): Promise<ResolvedOptions>`.
    Use `@clack/prompts` `intro`/`outro`, `text`, `select`, `confirm`, and `isCancel`.

    Wire into `src/index.ts` `main()`:
    1. `parseArgs(process.argv.slice(2))` → if `help`/`version`, handle; if `errors.length`, print and exit 1.
    2. `const opts = await runPrompts(parsed)`
    3. Log the resolved options (Wave 3 will hand them to the template engine). For now: `console.log(JSON.stringify(opts, null, 2))` so we can verify manually.

    Prompt tests: mock `@clack/prompts` (it exports replaceable functions; use `vi.mock`) and assert call order / arguments for each flag combination.
  </action>
