Every route file MUST start with `export const runtime = 'nodejs';` and wrap ALL handlers with `withApiKey`.

**1. `apps/api/src/app/api/v1/files/route.ts` — GET (list files, API-02):**
- Parse query params with `PaginationSchema`: `const url = new URL(req.url); const params = PaginationSchema.safeParse({ limit: url.searchParams.get('limit'), cursor: url.searchParams.get('cursor') });`
- Cursor-based pagination: if cursor provided, query `{ projectId: ctx.project._id, _id: { $lt: new Types.ObjectId(cursor) }, deletedAt: null }`, else `{ projectId: ctx.project._id, deletedAt: null }`
- Sort by `{ _id: -1 }`, limit to `params.data.limit + 1`
- Compute `hasMore = files.length > params.data.limit`, `nextCursor = hasMore ? files[params.data.limit - 1]._id.toString() : null`
- Return `{ files: files.slice(0, params.data.limit), nextCursor, hasMore }`
- Import `Types` from `mongoose` for ObjectId construction

**2. `apps/api/src/app/api/v1/files/[key]/route.ts` — GET, PATCH, DELETE (API-02):**

Note on key parameter: The file key contains slashes (`{projectId}/{routeSlug}/{nanoid}/{filename}`). The route param `[key]` will be URL-encoded by the client. Decode with `decodeURIComponent(params.key)`.

**GET:** `File.findOne({ key: decodedKey, projectId: ctx.project._id, deletedAt: null })` — 404 if not found. Return file object.

**PATCH:** Parse body with `UpdateFileMetadataSchema`. `File.findOneAndUpdate({ key: decodedKey, projectId: ctx.project._id, deletedAt: null }, { $set: { metadata: parsed.data.metadata } }, { new: true })` — 404 if not found. Return updated file.

**DELETE:**
1. `File.findOne({ key: decodedKey, projectId: ctx.project._id, deletedAt: null })` — 404 if not found
2. Delete from R2: `await r2Client.send(new DeleteObjectCommand({ Bucket: R2_BUCKET, Key: file.key }))` — import from `@aws-sdk/client-s3`
3. Soft delete: `File.findByIdAndUpdate(file._id, { $set: { deletedAt: new Date(), status: 'DELETED' } })`
4. Decrement storage: `UsageRecord.findOneAndUpdate({ userId: ctx.project.userId, period: new Date().toISOString().slice(0, 7) }, { $inc: { storageUsed: -file.size } }, { upsert: true })`
5. Return `NextResponse.json({ ok: true })`

**3. `apps/api/src/app/api/v1/projects/route.ts` — GET, POST (API-03):**

**GET:** `Project.find({ userId: ctx.project.userId })` — return `{ projects }`. Note: list all projects for the user (not just the one from the API key).

**POST:**
1. Parse body with `CreateProjectSchema`
2. Check tier limit: count projects `Project.countDocuments({ userId: ctx.project.userId })` — if `>= TIER_LIMITS[ctx.tier].maxProjects`, return `serializeError(new TierLimitError('projects'))`
3. Generate slug: `import { nanoid } from 'nanoid'; const slug = parsed.data.name.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '') + '-' + nanoid(6)`
4. Create: `Project.create({ name: parsed.data.name, slug, userId: ctx.project.userId })`
5. Return `{ project }` with status 201

**4. `apps/api/src/app/api/v1/projects/[id]/route.ts` — PATCH, DELETE (API-03):**

**PATCH:**
1. Validate `params.id` is valid ObjectId
2. Parse body with `UpdateProjectSchema`
3. `Project.findOneAndUpdate({ _id: params.id, userId: ctx.project.userId }, { $set: parsed.data }, { new: true })` — 404 if null
4. Return `{ project }`

**DELETE:**
1. Verify project exists and belongs to user: `Project.findOne({ _id: params.id, userId: ctx.project.userId })` — 404 if null
2. Check it is not the last project (optional safeguard)
3. Delete associated resources: `ApiKey.updateMany({ projectId: params.id }, { $set: { revokedAt: new Date() } })` — revoke all keys
4. `Project.deleteOne({ _id: params.id })`
5. Return `{ ok: true }`

All handlers wrapped in try/catch with `serializeError`.

**1. `apps/api/src/app/api/v1/projects/[id]/keys/route.ts` — GET, POST (API-04):**

**GET:** `ApiKey.find({ projectId: params.id, revokedAt: null }).select('-keyHash')` — return list with `keyPrefix` visible but no hash. Verify `params.id` project belongs to user via `ctx.project.userId`.

**POST (critical — API key creation):**
1. Parse body with `CreateApiKeySchema`
2. Verify the target project belongs to user: `Project.findOne({ _id: params.id, userId: ctx.project.userId })` — 404 if null
3. Check tier limit: `ApiKey.countDocuments({ projectId: params.id, revokedAt: null })` — if `>= TIER_LIMITS[ctx.tier].maxApiKeys`, return `serializeError(new TierLimitError('API keys'))`
4. Generate full key: `import { nanoid } from 'nanoid'; const isTest = parsed.data.isTest ?? false; const prefix = isTest ? API_KEY_PREFIX.test : API_KEY_PREFIX.live; const fullKey = prefix + nanoid(32);`
5. Hash with SHA256: `import { createHash } from 'node:crypto'; const keyHash = createHash('sha256').update(fullKey).digest('hex');`
6. Store: `ApiKey.create({ keyPrefix: fullKey.slice(0, 12), keyHash, name: parsed.data.name ?? 'Default', projectId: params.id, isTest })`
7. Return `{ apiKey: { id: doc._id, keyPrefix: doc.keyPrefix, name: doc.name, isTest: doc.isTest, createdAt: doc.createdAt }, key: fullKey }` with status 201
   - IMPORTANT: `key: fullKey` is returned ONLY at creation time. It is never stored and cannot be retrieved again.

**2. `apps/api/src/app/api/v1/keys/[keyId]/route.ts` — DELETE (revoke, API-04):**

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> files/:key | Untrusted: URL-encoded key could target another project's files |
| Client -> projects/:id/keys | Untrusted: project ID could belong to another user |
| Client -> API key creation | Sensitive: full key exposed once at creation time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Elevation | files/[key] | mitigate | All file queries scoped by `projectId` from withApiKey context |
| T-03-14 | Elevation | projects/[id]/keys | mitigate | Verify project.userId matches ctx.project.userId before key operations |
| T-03-15 | Info Disclosure | keys/POST | mitigate | Full key returned only at creation; DB stores SHA256 hash; keyPrefix shows first 12 chars only |
| T-03-16 | Tampering | files/[key]/DELETE | mitigate | Atomic $inc with negative value for storage decrement; soft delete prevents double-decrement |
| T-03-17 | Spoofing | routers/[routerId] | mitigate | Router lookup verifies project ownership chain before any mutation |
</threat_model>
