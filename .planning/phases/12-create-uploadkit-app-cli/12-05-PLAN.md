  <action>
    `+server.ts` POST handler:
    - Validates body: `{ filename: string, contentType: string }` with zod (or hand-rolled guard to avoid the dep).
    - Generates an R2 key: `${crypto.randomUUID()}-${filename}`.
    - Uses `@aws-sdk/s3-request-presigner` `getSignedUrl` with `PutObjectCommand`, `ContentType` explicitly set (per CLAUDE.md — signature mismatches cause 403).
    - Returns `json({ url, key, method: 'PUT' })`.
    - Reads credentials from `env` — `R2_*` values. If missing: returns `json({ error: 'Set R2_* env vars in .env.local' }, 503)`.

    `+page.svelte` (Svelte 5 runes):
    ```svelte
    <script lang="ts">
      let status = $state<'idle' | 'uploading' | 'done' | 'error'>('idle');
      let file = $state<File | null>(null);

      async function upload() {
        if (!file) return;
        status = 'uploading';
        try {
          const sign = await fetch('/api/sign', {
            method: 'POST',
            headers: { 'content-type': 'application/json' },
            body: JSON.stringify({ filename: file.name, contentType: file.type }),
          }).then((r) => r.json());
          if (sign.error) throw new Error(sign.error);
          const put = await fetch(sign.url, { method: 'PUT', body: file, headers: { 'content-type': file.type } });
          status = put.ok ? 'done' : 'error';
        } catch {
          status = 'error';
        }
      }
    </script>

    <main>
      <h1>UploadKit — SvelteKit</h1>
      <input type="file" onchange={(e) => (file = (e.currentTarget as HTMLInputElement).files?.[0] ?? null)} />
      <button onclick={upload} disabled={!file || status === 'uploading'}>Upload</button>
      <p>Status: {status}</p>
    </main>
    ```

    Smoke test asserts the scaffolded project contains `+server.ts` and a `+page.svelte` that posts to `/api/sign`.
