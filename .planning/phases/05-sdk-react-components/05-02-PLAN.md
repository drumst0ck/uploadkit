    **useDragState hook (RESEARCH.md Pattern 3 — counter technique):**

    Create `packages/react/src/hooks/use-drag-state.ts`:
    - Uses integer counter state (not boolean) to fix child element drag event flickering
    - Exports `useDragState()` returning `{ isDragging, handlers: { onDragEnter, onDragLeave, onDragOver, onDrop } }`
    - `onDragEnter`: preventDefault, increment counter
    - `onDragLeave`: preventDefault, decrement counter
    - `onDragOver`: preventDefault (required to allow drop)
    - `onDrop`: preventDefault, set counter to 0, return files from `e.dataTransfer.files`
    - `isDragging = counter > 0`
    - Accept an `onFiles: (files: File[]) => void` callback parameter for drop handler

    **useAutoDismiss hook (per D-08):**

    Create `packages/react/src/hooks/use-auto-dismiss.ts`:
    - `useAutoDismiss(items: T[], duration = 5000)` — returns items that auto-remove after duration
    - Uses useEffect + setTimeout to remove each item after the specified duration
    - Returns `[visibleItems, addItem, removeItem]`

    **UploadDropzone (REACT-04, D-07, D-08, D-05):**

    Create `packages/react/src/components/upload-dropzone.tsx`:

    Props type `UploadDropzoneProps`:
    ```typescript
    type UploadDropzoneProps = {
      route: string;
      accept?: string[];
      maxSize?: number;
      maxFiles?: number;
      metadata?: Record<string, unknown>;
      onUploadComplete?: (results: UploadResult[]) => void;
      onUploadError?: (error: Error) => void;
      disabled?: boolean;
      className?: string;
      appearance?: Partial<Record<'container' | 'label' | 'icon' | 'fileItem' | 'progressBar' | 'button', string>>;
    };
    ```

    Implementation:
    1. Use `useUploadKitContext()` to get client directly (not useUploadKit — dropzone manages multiple files with individual state)
    2. Maintain internal file queue state: `Array<{ id: string; file: File; status: UploadStatus; progress: number; result?: UploadResult; error?: Error; abortController?: AbortController }>`
    3. Use `useDragState` for drag-and-drop handlers
    4. Hidden `<input type="file" multiple ref={inputRef} />` for click-to-browse fallback
    5. On files received (drop or input change):
       a. Validate each file: check type against `accept`, check size against `maxSize`, check count against `maxFiles`
       b. Rejected files -> add to rejection list (inline error toast per D-08)
       c. Accepted files -> add to queue with 'idle' status, then upload each sequentially (or in parallel up to 3 concurrent)
    6. For each uploading file: create AbortController, call `client.upload({ file, route, metadata, onProgress, signal })`, update individual file progress
    7. When all files complete -> call `onUploadComplete` with results array

    **File rejection error toasts (D-08):**
    - Use `useAutoDismiss` for rejection messages
    - Format: "{filename} rejected -- {reason}" (e.g., "photo.exe rejected -- .exe files not allowed" or "large.zip rejected -- exceeds 4MB limit")
    - Red text (`.uk-error-toast`), auto-dismiss after 5s

    **Multi-file list (D-07):**
    Below the dropzone, render a stacked list of queued files:
    ```tsx
    <div className="uk-file-list">
      {files.map(f => (

          <span className="uk-file-item__name" title={f.file.name}>{f.file.name}</span>
          <span className="uk-file-item__size">{formatBytes(f.file.size)}</span>
          {f.status === 'uploading' && (
            <div className={mergeClass('uk-progress', appearance?.progressBar)}>
              <div className="uk-progress__bar" style={{ width: `${f.progress}%` }} role="progressbar" ... />
            </div>
          )}
          {f.status === 'idle' || f.status === 'uploading' ? (
            <button className="uk-file-item__remove" onClick={() => removeFile(f.id)} aria-label={`Remove ${f.file.name}`}>
              {/* X icon SVG */}
            </button>
          ) : null}
          {f.status === 'success' && <span>check icon</span>}
          {f.status === 'error' && <span className="uk-file-item__error">error icon</span>}
        </div>
      ))}
    </div>
    ```

    **Dropzone render structure:**
    ```tsx
    <div
      ref={ref}
      className={mergeClass(`uk-dropzone ${isDragging ? '' : ''}`, appearance?.container)}
      data-dragging={isDragging}
      {...dragHandlers}
      onClick={handleClick}
      role="button"
      tabIndex={0}
      aria-label="Drop files here or click to browse"
      onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}
    >
      <div className={mergeClass('uk-dropzone__icon', appearance?.icon)} dangerouslySetInnerHTML={{ __html: getUploadIcon() }} />
      <p className={mergeClass('uk-dropzone__label', appearance?.label)}>
        <strong>Drop files here</strong> or click to browse
      </p>
      {accept && <p className="uk-dropzone__hint">{accept.join(', ')}</p>}
    </div>
    {/* Error toasts */}
    {rejections.map(r => <div key={r.id} className="uk-error-toast">{r.message}</div>)}
    {/* File list */}
    {files.length > 0 && <div className="uk-file-list">...</div>}
    <input ref={inputRef} type="file" multiple={maxFiles !== 1} hidden accept={accept?.join(',')} ... />
    ```
