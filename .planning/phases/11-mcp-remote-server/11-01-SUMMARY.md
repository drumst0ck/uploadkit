## Deviations from Plan

### [Rule 1 - Bug] Replaced `sessionIdGenerator: undefined` with omitted key
- **Found during:** Task 3 typecheck.
- **Issue:** `exactOptionalPropertyTypes: true` rejects explicit `undefined` for an optional property typed `() => string`.
- **Fix:** Passed `{}` to `WebStandardStreamableHTTPServerTransport`. The SDK's stateless mode triggers when `sessionIdGenerator` is absent, producing identical behaviour.
- **Files:** `apps/api/src/app/api/v1/mcp/route.ts`
- **Commit:** 6f3d636

### [Rule 3 - Blocking] Stubbed `server-only` in vitest
- **Found during:** Task 4 test run.
- **Issue:** `apps/api/src/lib/mcp-server.ts` imports `server-only` (correct Next.js guard). Vitest cannot resolve it outside a Next build.
- **Fix:** Added `apps/api/__tests__/stubs/server-only.ts` (empty module) and aliased `server-only` to it in `vitest.config.ts`. Production build still enforces the server boundary via the real module Next ships.
- **Commit:** 4fdced2

### [Rule 1 - Bug] Duplicate shebang in stdio bin
- **Found during:** Task 2 stdio smoke test.
- **Issue:** Source file kept `#!/usr/bin/env node` while tsup's `banner: { js: '#!/usr/bin/env node' }` also injects one — Node rejected the duplicate line.
- **Fix:** Removed the in-source shebang; tsup banner remains the single source.
- **Commit:** 462744a

### [Rule 3 - Blocking] `types: ["node"]` in packages/mcp/tsconfig.json
- **Found during:** Task 2 `tsup` DTS build.
- **Issue:** New bootstrap uses `console.error` + `process.exit`; existing tsconfig did not expose Node ambient types.
- **Fix:** Added `"types": ["node"]` to the local tsconfig.
- **Commit:** 462744a

### Package.json exports map
- **Decision:** `@uploadkitdev/mcp-core/package.json` exports `./dist/index.js` (ESM, default since `"type": "module"`) and `./dist/index.cjs` (CJS). The plan wrote `./dist/index.mjs` which doesn't match tsup's output filenames under `"type": "module"` packages. Adjusted to match tsup reality.
