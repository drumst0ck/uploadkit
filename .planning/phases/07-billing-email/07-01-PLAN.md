<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard -> Stripe API | Server-side only; secret key never in browser |
| Dashboard server action -> Stripe Checkout | Redirect URL; user leaves UploadKit temporarily |
| API billing routes -> Stripe API | Server-side; validates userId matches session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | billing/checkout | mitigate | Validate userId from auth session, never accept userId from client body in server actions |
| T-07-02 | Tampering | billing/checkout | mitigate | Validate priceId against known env var values; reject unknown price IDs with 400 |
| T-07-03 | Elevation of Privilege | billing/portal | mitigate | Look up stripeCustomerId from DB by authenticated userId; never accept customerId from client |
| T-07-04 | Information Disclosure | billing/page.tsx | mitigate | Invoice data fetched server-side; only user's own invoices shown (scoped by their stripeCustomerId) |
</threat_model>
