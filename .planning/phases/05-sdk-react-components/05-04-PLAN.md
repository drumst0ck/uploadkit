    The factory returns wrapped components where `route` is constrained to `keyof TRouter & string`:

    ```typescript
    export function generateReactHelpers<TRouter extends FileRouter>() {
      // Create typed wrapper types where route is constrained
      type TypedButtonProps = Omit<UploadButtonProps, 'route'> & { route: keyof TRouter & string };
      type TypedDropzoneProps = Omit<UploadDropzoneProps, 'route'> & { route: keyof TRouter & string };
      type TypedModalProps = Omit<UploadModalProps, 'route'> & { route: keyof TRouter & string };

      return {
        UploadButton: UploadButton as React.ComponentType<TypedButtonProps>,
        UploadDropzone: UploadDropzone as React.ComponentType<TypedDropzoneProps>,
        UploadModal: UploadModal as React.ComponentType<TypedModalProps>,
        useUploadKit: useUploadKit as (route: keyof TRouter & string) => ReturnType<typeof useUploadKit>,
      };
    }
    ```

    This approach:
    - Does NOT create wrapper components (no extra React elements in the tree)
    - Uses TypeScript assertion to narrow the `route` prop type
    - The `keyof TRouter & string` constraint provides autocomplete for route names
    - The `Omit + &` pattern replaces only the `route` type while keeping all other props

       - `.uk-file-item__remove:focus-visible` — same outline
       - `.uk-modal:focus-visible` — remove default outline (dialog has its own focus management)
       - Add any missing `:focus-visible` rules

    2. **aria-labels** — Verify every interactive element has appropriate aria attributes:
       - UploadButton: `aria-busy={isUploading}`, `aria-label` reflecting state
       - UploadDropzone: `role="button"`, `tabIndex={0}`, `aria-label="Drop files here or click to browse"`
       - UploadDropzone file items: remove button has `aria-label="Remove {filename}"`
       - UploadModal dialog: `aria-label={title || 'Upload files'}`
       - FileList delete buttons: `aria-label="Delete {filename}"`
       - Progress bars: `role="progressbar"`, `aria-valuenow`, `aria-valuemin={0}`, `aria-valuemax={100}`

    3. **aria-live for progress announcements** — Add an `aria-live="polite"` region that announces upload status changes:
       - In UploadButton: a visually hidden span that announces "{progress}% uploaded" during upload, "Upload complete" on success, "Upload failed" on error
       - In UploadDropzone: a visually hidden span announcing "{n} files uploading, {m} complete"
       - Add `.uk-sr-only` class to styles.css: `position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0 0 0 0); white-space: nowrap; border: 0`

    4. **Reduced motion** — Verify the `prefers-reduced-motion` media query in styles.css disables:
       - All `transition-duration` set to `0.01ms`
       - All `animation-duration` set to `0.01ms`
       - This applies to all `.uk-*` elements
       - Modal enter/exit animation respects reduced motion

    5. **Keyboard navigation** — Verify:
       - UploadDropzone responds to Enter and Space key presses (opens file picker)
       - UploadButton responds to Enter and Space (native button behavior)
       - UploadModal closes on ESC (native dialog behavior)
       - Tab order is logical within all components

    6. **Color contrast** — Verify CSS variables meet WCAG AA (4.5:1 for text, 3:1 for UI components):

       - Dark mode: `--uk-text-secondary: #888888` on `--uk-bg: #0a0a0a` — ratio ~8:1 (passes)
