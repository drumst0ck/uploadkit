**1. Create `apps/api/src/lib/errors.ts` — Stripe-style error serializer (per D-07):**

The error format distinguishes retryable from non-retryable errors (per UPLD-07 server contract):
- 5xx and 429 responses are retryable — the SDK will retry these automatically
- 4xx responses (except 429) are non-retryable — the SDK surfaces these as user errors immediately

```typescript
import { UploadKitError } from '@uploadkit/shared';
import { NextResponse } from 'next/server';

const ERROR_TYPE_MAP: Record<number, string> = {
  400: 'invalid_request',
  401: 'authentication_error',
  403: 'invalid_request',
  404: 'invalid_request',
  422: 'invalid_request',
  429: 'rate_limit_error',
  500: 'api_error',
};

export function serializeError(err: unknown): NextResponse {
  if (err instanceof UploadKitError) {
    return NextResponse.json(
      {
        error: {
          type: ERROR_TYPE_MAP[err.statusCode] ?? 'api_error',
          code: err.code,
          message: err.message,
          ...(err.suggestion ? { suggestion: err.suggestion } : {}),
        },
      },
      { status: err.statusCode },
    );
  }
  console.error('Unhandled error:', err);
  return NextResponse.json(
    { error: { type: 'api_error', code: 'INTERNAL_ERROR', message: 'An unexpected error occurred' } },
    { status: 500 },
  );
}
```

Also add a `serializeValidationError(zodError)` helper that maps Zod v4 `safeParse` failures to the Stripe format with `details` field (per D-08):
```typescript
export function serializeValidationError(zodError: z.ZodError): NextResponse {
  return NextResponse.json(
    {
      error: {
        type: 'invalid_request',
        code: 'VALIDATION_ERROR',
        message: 'Invalid request body',
        details: zodError.flatten().fieldErrors,
      },
    },
    { status: 400 },
  );
}
```

**2. Create `apps/api/src/lib/with-api-key.ts` — Auth wrapper (per D-02):**

Export `ApiContext` interface:
```typescript
export interface ApiContext {
  apiKey: IApiKey;
  project: IProject;
  tier: Tier;
}
```

Export `withApiKey(handler, useUploadLimit?)` HOF:
- Extract `Authorization: Bearer <token>` header. If missing/malformed, return `serializeError(new UnauthorizedError())`
- Call rate limiter FIRST (Upstash HTTP, no DB). Use `uploadRatelimit` if `useUploadLimit=true`, else `ratelimit`. Key by `apikey:<first 20 chars of token>`. On failure, return `serializeError(new RateLimitError(retryAfterSeconds))`
- `await connectDB()`
- Hash incoming token with SHA256: `import { createHash } from 'node:crypto'; const hash = createHash('sha256').update(token).digest('hex');`
- Look up `ApiKey.findOne({ keyHash: hash, revokedAt: null }).populate('projectId')`
- If not found, return `serializeError(new UnauthorizedError())`
- Fire-and-forget `lastUsedAt` update: `void ApiKey.updateOne({ _id: apiKeyDoc._id }, { $set: { lastUsedAt: new Date() } })`
- Look up subscription: `Subscription.findOne({ userId: project.userId })` — default tier to `'FREE'` if none
- Call inner handler with `(req, { apiKey: apiKeyDoc, project, tier }, params)`

The handler type signature:
```typescript
type Handler = (req: NextRequest, ctx: ApiContext, params?: Record<string, string | string[]>) => Promise<NextResponse>;
```

The returned wrapper must accept Next.js App Router signature: `(req: NextRequest, segmentData: { params: Promise<Record<string, string | string[]>> })`. Note: Next.js 16 uses `Promise<Params>` — the wrapper must `await segmentData.params` before passing to handler.
