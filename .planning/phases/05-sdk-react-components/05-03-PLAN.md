    2. `useEffect` watching `open`:

    3. Handle ESC via `onCancel` event on dialog:
       - `e.preventDefault()` to stop native dialog close (per Pitfall 4)
       - Call `onClose()` to let React state drive
    4. Handle click-outside via `onClick` on dialog element:
       - Check if `e.target === dialogRef.current` (clicked the backdrop, not content)
       - If so, call `onClose()`
    5. Render an `UploadDropzone` inside the modal content area (composition, not reimplementation):
       - Pass through `route`, `accept`, `maxSize`, `maxFiles`, `metadata`, `onUploadComplete`, `onUploadError`
       - Pass appropriate `appearance` sub-props

    Render structure:
    ```tsx
    <dialog
      ref={dialogRef}
      className={mergeClass('uk-modal', appearance?.modal)}
      onCancel={handleCancel}
      onClick={handleBackdropClick}
      aria-label={title || 'Upload files'}
    >
      <div className={mergeClass('uk-modal__content', appearance?.content)}>
        {title && <h2 className="uk-modal__title">{title}</h2>}
        <UploadDropzone
          route={route}
          accept={accept}
          maxSize={maxSize}
          maxFiles={maxFiles}
          metadata={metadata}
          onUploadComplete={onUploadComplete}
          onUploadError={onUploadError}

            icon: appearance?.icon,

            button: appearance?.button,
    ```

    **useThumbnail hook (D-09, RESEARCH.md Pattern 5):**

    Create `packages/react/src/hooks/use-thumbnail.ts`:

    Export `useThumbnail(file: File | null)`:
    1. Returns `{ thumbnailUrl: string | null; isGenerating: boolean }`
    2. Uses `useState` for thumbnailUrl and isGenerating
    3. `useEffect` on `file`:
       a. If file is null, return
       b. If file is image/* — use `URL.createObjectURL(file)` directly as thumbnailUrl (no canvas needed for display; canvas only needed for resize which is optional). Set thumbnailUrl, isGenerating false.
       c. If file is video/* — create detached `<video>`, set `src = URL.createObjectURL(file)`, `muted = true`, `playsInline = true`. Listen for `loadedmetadata` -> set `currentTime = 0`. Listen for `seeked` -> create canvas (160px wide, proportional height), `drawImage(video)`, `canvas.toDataURL('image/jpeg', 0.8)` -> set thumbnailUrl. `URL.revokeObjectURL` after.
       d. Otherwise — return null (no thumbnail, component will show type icon)
    4. Cleanup function: `URL.revokeObjectURL` for any active object URLs (per Pitfall 5)
    5. Use `IntersectionObserver` pattern: only generate thumbnail when a ref element is visible (per CONTEXT.md specific idea). Accept an optional `containerRef` parameter. If provided, observe it and only start generation when visible. If not provided, generate immediately.

## Trust Boundaries

|----------|-------------|
| User files -> Canvas API | Untrusted file content drawn to canvas for thumbnail |
| UploadResult.url -> img src | Server-provided URLs rendered in img tags |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | T (Tampering) | FilePreview img src | mitigate | UploadResult.url comes from UploadKit API (trusted); File thumbnails use createObjectURL (same-origin blob) — no cross-origin risk |
| T-05-08 | D (Denial of Service) | useThumbnail | mitigate | IntersectionObserver ensures thumbnails only generate for visible elements; revokeObjectURL prevents memory accumulation |
| T-05-09 | I (Information Disclosure) | dialog element | accept | Modal content is visible when open — by design. Dialog prevents interaction with background content via native inert behavior |
