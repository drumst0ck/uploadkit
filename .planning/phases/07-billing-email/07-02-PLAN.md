Create `apps/api/src/app/api/v1/webhooks/stripe/route.ts`:

1. `export const runtime = 'nodejs';` (required for crypto in constructEvent).

2. POST handler:
   - Read raw body with `req.text()` (NEVER `req.json()` — breaks HMAC, Pitfall 1).
   - Get `stripe-signature` header.
   - Call `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`. Catch errors and return 400 with `{ error: 'Invalid signature' }`.
   - `await connectDB()`.
   - Switch on `event.type` (handler map pattern per Claude's discretion):

3. Event handlers:

   **`checkout.session.completed`:**
   - Extract `userId` from `session.metadata?.userId`. Return early if missing.
   - Retrieve full subscription: `stripe.subscriptions.retrieve(session.subscription as string)`.
   - Extract `priceId` from `sub.items.data[0]?.price.id`.
   - Map priceId to tier: compare against `STRIPE_PRO_PRICE_ID` and `STRIPE_TEAM_PRICE_ID` env vars. Default to 'PRO' if unknown.
   - Upsert Subscription: `Subscription.findOneAndUpdate({ userId }, { $set: { stripeSubscriptionId: sub.id, stripePriceId: priceId, stripeCustomerId: session.customer as string, tier, status: 'ACTIVE', currentPeriodStart: new Date(sub.current_period_start * 1000), currentPeriodEnd: new Date(sub.current_period_end * 1000), cancelAtPeriodEnd: sub.cancel_at_period_end } }, { upsert: true, new: true })`.
   - Idempotent via upsert (Pitfall 3).

   **`customer.subscription.updated`:**
   - Extract subscription object from event.data.object.
   - Look up by `stripeSubscriptionId`.
   - Map new priceId to tier. Update `tier`, `status` (map Stripe status: 'active' -> 'ACTIVE', 'past_due' -> 'PAST_DUE', 'canceled' -> 'CANCELED', 'trialing' -> 'TRIALING'), `currentPeriodStart`, `currentPeriodEnd`, `cancelAtPeriodEnd`.

   **`customer.subscription.deleted`:**
   - Find by `stripeSubscriptionId`.
   - Set `tier: 'FREE'`, `status: 'CANCELED'`, `cancelAtPeriodEnd: false`. Clear `stripeSubscriptionId` and `stripePriceId`.

   **`invoice.paid`:**
   - Extract customer ID. Find Subscription by `stripeCustomerId`.
   - If subscription status was PAST_DUE, update to ACTIVE (payment recovered).

   **`invoice.payment_failed`:**
   - Extract customer ID. Find Subscription by `stripeCustomerId`.
   - Set `status: 'PAST_DUE'`.

4. Default case: Return `{ received: true }` with 200 for unrecognized events (per Stripe best practice — prevents retry loops).

5. Return `NextResponse.json({ received: true })` for all handled events.

Per D-02: Webhook handler lives at `POST /api/v1/webhooks/stripe`, consistent with existing QStash DLQ webhook pattern.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe -> webhook handler | Untrusted HTTP; must verify HMAC signature |
| Upload client -> request route | Tier enforcement boundary |
| Upload complete -> Stripe Meters | Outbound API; identifier prevents replay |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Spoofing | webhooks/stripe | mitigate | `stripe.webhooks.constructEvent()` with STRIPE_WEBHOOK_SECRET; reject invalid signatures with 400 |
| T-07-06 | Tampering | webhooks/stripe | mitigate | Use `req.text()` (never `req.json()`) to preserve raw body for HMAC verification |
| T-07-07 | Repudiation | upload/complete | mitigate | MeterEvent `identifier` field (file._id) prevents double-counting on webhook retry |
| T-07-08 | Denial of Service | webhooks/stripe | accept | Stripe retries on non-200; returning 200 for all events prevents retry storms |
| T-07-09 | Elevation of Privilege | webhooks/stripe | mitigate | Tier derived from priceId->env var mapping, never from client-supplied data; unknown priceIds default to 'PRO' (not 'ENTERPRISE') |
| T-07-10 | Tampering | upload/request | mitigate | Tier resolved from DB (withApiKey), not from request; client cannot claim a different tier |
</threat_model>
