    **rest-api.mdx** — "REST API Overview"
    - Base URL: `https://api.uploadkit.dev/api/v1`
    - Authentication: API key via `x-api-key` header
    - Response format: JSON, consistent error shape
    - Rate limiting: sliding window, 429 response with Retry-After
    - Pagination: cursor-based with `limit` and `cursor` params
    - Links to each endpoint section

    **authentication.mdx** — "Authentication"
    - API key types: `uk_live_` (production) and `uk_test_` (test mode)
    - Header format: `x-api-key: uk_live_xxxxx`
    - Creating keys (link to dashboard docs)
    - Key security (hashed with SHA256, plaintext shown only at creation)
    - Example request with auth header
    - 401 response format

    **upload.mdx** — "Upload Endpoints" — Each endpoint as a section with its own curl example:
    - POST /api/v1/upload/request — Request presigned URL. Params from UploadRequestSchema. Response: { fileId, uploadUrl, key, cdnUrl }
    - POST /api/v1/upload/complete — Confirm upload. Params from UploadCompleteSchema. Response: file object.
    - POST /api/v1/upload/multipart/init — Init multipart. Params from MultipartInitSchema. Response: { fileId, uploadId, key, parts: [{ partNumber, uploadUrl }] }
    - POST /api/v1/upload/multipart/complete — Complete multipart. Params from MultipartCompleteSchema.
    - POST /api/v1/upload/multipart/abort — Abort multipart. Params from MultipartAbortSchema.
    - For each: method, URL, request body table (field, type, required, description), response body, curl example, error cases.

    **files.mdx** — "Files Endpoints":
    - GET /api/v1/files — List files (paginated). Query params from PaginationSchema.
    - GET /api/v1/files/:key — Get single file.
    - PATCH /api/v1/files/:key — Update metadata. Body from UpdateFileMetadataSchema.
    - DELETE /api/v1/files/:key — Delete file (R2 + DB).
    - For each: curl example, response shape.

    **projects.mdx** — "Projects Endpoints":
    - GET /api/v1/projects — List projects.
    - POST /api/v1/projects — Create project. Body from CreateProjectSchema.
    - PATCH /api/v1/projects/:id — Update. Body from UpdateProjectSchema.
    - DELETE /api/v1/projects/:id — Delete (cascades files, keys, routes).

    **webhooks.mdx** — "Webhooks":
    - onUploadComplete webhook (fired via QStash after file upload)
    - Webhook payload shape (file object + metadata)
    - Webhook URL configuration (per file route in dashboard or via API)
    - Verification (QStash signature)
    - DLQ handling (dead letter queue for failed deliveries)
    - Retry behavior

    **errors.mdx** — "Error Codes":
    - Error response format: `{ error: { code: string, message: string, suggestion?: string } }`
    - Error codes table (read from packages/shared/src/errors.ts if error codes are defined there): INVALID_API_KEY, RATE_LIMIT_EXCEEDED, FILE_TOO_LARGE, FILE_TYPE_NOT_ALLOWED, UPLOAD_NOT_FOUND, PROJECT_NOT_FOUND, TIER_LIMIT_EXCEEDED, etc.
    - Each code: HTTP status, description, common cause, fix suggestion
    - Rate limit headers: X-RateLimit-Limit, X-RateLimit-Remaining, Retry-After

      | `f()` chain | `satisfies FileRouter` object |
      | `useUploadThing()` | `useUploadKit()` |
      | `<UploadButton endpoint="..." />` | `<UploadButton route="..." />` |
      | `<UploadDropzone endpoint="..." />` | `<UploadDropzone route="..." />` |

      2. Update file router syntax
      3. Update component imports and props (endpoint -> route)
      4. Update environment variables
      5. Test upload flow
    - What's new in UploadKit (BYOS, premium components, better theming)
