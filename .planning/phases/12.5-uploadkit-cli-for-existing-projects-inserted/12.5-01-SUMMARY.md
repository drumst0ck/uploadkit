# Phase 12.5 Plan 01: Package Skeleton + Subcommand Router Summary

Scaffolded `packages/uploadkit-cli/` (publishing as unscoped npm `uploadkit`) with the Phase 12 toolchain — tsup ESM build with shebang banner, vitest, strict TypeScript — and wired a minimal CLI surface (`init` / `add` / `restore` / `--version` / `--help`) whose command handlers are placeholders ready for plans 04–07 to fill.

## What was built

### Task 1 — Package skeleton

Mirrored the layout of `packages/create-uploadkit-app/` and swapped names/bins. `package.json` declares the unscoped name `uploadkit` (D-01), `bin.uploadkit = "./dist/index.js"`, `type: module`, `engines.node >=20`, and the deps locked by `@latest`. `tsup.config.ts` emits a single ESM bundle with `#!/usr/bin/env node` banner; `vitest.config.ts` uses the node environment; `tsconfig.json` extends `@uploadkitdev/config/typescript/library` with `strict` (inherited).

`pnpm-workspace.yaml` already globs `packages/*` so no change was needed (the plan flagged "likely no change needed" and that was correct).

### Task 2 — Entrypoint + args parser + version

- `src/index.ts`: thin entry. Parses argv, prints VERSION for `--version`, prints help for `--help` or no command, otherwise calls `dispatch`. Auto-run guarded by `UPLOADKIT_CLI_SKIP_MAIN` so tests can import `main` without side effects.
- `src/args.ts`: `parseArgs(argv)` wraps `mri` and returns `{ command, positional, flags }`. Flags bag covers every declared flag across subcommands (`yes`, `skipInstall`, `target`, `timestamp`, plus `version`/`help`), keeping the parser generic — each command reads only what it needs.
- `src/version.ts`: exports `VERSION = '0.0.0'`. A later plan (per Phase 12 pattern) replaces this at publish time.
- 8 unit tests in `__tests__/args.test.ts` lock the shape: subcommand, positional, short aliases, `--target`, `--timestamp`, `--skip-install`, and default-false booleans.

### Task 3 — Subcommand router

- `commands/index.ts`: exports a frozen `handlers` map and a `dispatch(parsed)` function. Unknown command → stderr + exit code 1. Missing command → stderr + exit code 1. `dispatch` returns a number instead of calling `process.exit` directly, which makes the test in `dispatch.test.ts` trivial (spy the handlers, assert the return code).
- `commands/{init,add,restore}.ts`: each exports `run(parsed)` that prints a cyan `[uploadkit] <cmd> — not implemented yet (plan XX)` and returns 0. Plans 04/05/06/07 replace these bodies.
- `commands/help.ts`: `printHelp()` + `HELP_TEXT` listing the D-08 subcommand surface and global flags.
- 6 tests in `__tests__/dispatch.test.ts` cover routing (init/add/restore), unknown command, missing command, and the canonical handler set.

## Deviations from Plan

None of substance. Two minor choices inside Claude's discretion:

1. Added `magicast@latest` to `dependencies` now (plan 03 needs it) rather than waiting for plan 03 to modify the manifest. This keeps install/lockfile churn in one commit instead of two.
2. `dispatch` returns an exit code rather than calling `process.exit` itself — made the unit tests cleaner (no `process.exit` spy needed) and matches the pattern already used by `create-uploadkit-app/src/index.ts::main`.

Neither affects the behavioral contract: all `<done>` criteria met.
