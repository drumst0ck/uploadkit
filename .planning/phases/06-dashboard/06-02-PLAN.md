## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/internal/projects | Session-based auth, project data scoped to userId |
| client -> /api/internal/overview | Session-based auth, usage data scoped to userId |
| URL params ([slug]) -> DB query | Slug from URL used in DB query with userId scoping |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Spoofing | /api/internal/projects POST | mitigate | auth() check before any DB operation. userId from session, never from request body. |
| T-06-06 | Tampering | Project slug in URL | mitigate | Project lookup always includes `userId: session.user.id` to prevent cross-user access via slug guessing. |
| T-06-07 | Information Disclosure | Overview metrics | mitigate | All queries filter by userId from server session. No cross-user data leakage. |
| T-06-08 | Denial of Service | Project creation | mitigate | Validate name length 1-50 chars. Rate limiting inherited from Next.js middleware. |
</threat_model>
