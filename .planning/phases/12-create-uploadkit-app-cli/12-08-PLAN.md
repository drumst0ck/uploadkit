<task type="auto">
  <name>Task 1: Template dep pinning + Changeset + prepublishOnly hook</name>
  <files>
    packages/create-uploadkit-app/scripts/pin-template-deps.mjs,
    packages/create-uploadkit-app/package.json,
    .changeset/create-uploadkit-app-initial.md
  </files>
  <action>
    Script `scripts/pin-template-deps.mjs`:
    - Iterates each directory under `../templates/`.
    - For each `package.json`, scans `dependencies` + `devDependencies`.
    - For every entry matching `@uploadkitdev/*`:
      - Reads that workspace package's `package.json` `version`.
      - Rewrites the dep to `^<version>`.
    - Writes the file back with trailing newline.
    - Also normalises any `"latest"` version strings in deps/devDeps to
      an actual resolved version by running
      `npm view <name> version --json` (batched, cached in-memory).
    - Idempotent; logs each rewrite.

    Update `package.json`:
    - Add `"prepublishOnly": "node scripts/pin-template-deps.mjs"`.
    - Add `"scripts.smoke": "node scripts/smoke-test.mjs"` (used in 12-08 task 2).

    Create `.changeset/create-uploadkit-app-initial.md` adding the
    package at `minor` to `create-uploadkit-app`.

    Note: Because this runs at `prepublishOnly`, local dev continues to
    use `workspace:*` / `latest`, and only the tarball sent to npm has
    pinned versions. This avoids polluting git with version churn.
  </action>
  <verify>
    <automated>node packages/create-uploadkit-app/scripts/pin-template-deps.mjs --dry-run</automated>
  </verify>
  <done>Dry-run prints planned rewrites for every template without touching disk; live run rewrites correctly.</done>
</task>

<task type="auto">
  <name>Task 2: Smoke test script + CI workflow</name>
  <files>
    packages/create-uploadkit-app/scripts/smoke-test.mjs,
    .github/workflows/smoke-create-uploadkit-app.yml,
    .github/workflows/release.yml
  </files>
  <action>
    Script `scripts/smoke-test.mjs`:
    1. `pnpm --filter create-uploadkit-app build`
    2. `pnpm --filter create-uploadkit-app pack` → produces a `.tgz`
    3. For each template in `['next', 'sveltekit', 'remix', 'vite']`:
       a. Create `os.tmpdir()/cka-smoke-<rand>/<template>`
       b. Run `npx --yes <tgz-path> <tmpdir>/demo --template <template> --pm pnpm --yes --no-git`
       c. `cd <tmpdir>/demo && pnpm install --prefer-offline && pnpm build`
       d. Assert build succeeded (exit code 0)
       e. Time the full scaffold step — fail if >90s (gives 50% headroom over the 60s target on slower CI runners)
    4. Print a summary table.

    CI workflow `smoke-create-uploadkit-app.yml`:
    - Triggers: push to master, pull_request touching `packages/create-uploadkit-app/**` or `packages/react/**` or `packages/next/**`
    - Matrix: `os: [ubuntu-latest, windows-latest]` (Windows validates D-01 path handling)
    - Steps: checkout → setup Node 20 → setup pnpm → install → run smoke script
    - Upload any scaffolded-project logs on failure

    Update existing `release.yml`:
    - Ensure `create-uploadkit-app` is in the published workspace set
    - Add a job dependency: release job requires `smoke-create-uploadkit-app` green
  </action>
  <verify>
    <automated>pnpm --filter create-uploadkit-app build &amp;&amp; node packages/create-uploadkit-app/scripts/smoke-test.mjs</automated>
  </verify>
  <done>
    - Local smoke run builds all 4 templates in &lt;90s (warm cache)
    - CI workflow green on both Linux and Windows
    - Release workflow publishes `create-uploadkit-app` to npm and the binary is reachable via `npx create-uploadkit-app@latest`
  </done>
</task>
