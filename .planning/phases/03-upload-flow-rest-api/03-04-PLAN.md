**1. Create `apps/api/src/app/api/v1/logs/route.ts` — GET (API-07):**

Add `export const runtime = 'nodejs';`

Export `GET` wrapped with `withApiKey(handler)`.

Handler steps:
1. Parse query params with `LogsQuerySchema`: `const url = new URL(req.url); const parsed = LogsQuerySchema.safeParse({ since: url.searchParams.get('since'), limit: url.searchParams.get('limit') });`
2. On parse failure, return `serializeValidationError(parsed.error)`
3. Query: `File.find({ projectId: ctx.project._id, createdAt: { $gt: parsed.data.since } }).sort({ createdAt: -1 }).limit(parsed.data.limit).lean()`
4. Return `NextResponse.json({ logs: files.map(f => ({ id: f._id, key: f.key, name: f.name, size: f.size, type: f.type, status: f.status, createdAt: f.createdAt, updatedAt: f.updatedAt })), count: files.length })`

This endpoint is designed for 5-second polling from the dashboard — the `since` param is the timestamp of the last received event.

**2. Create `apps/api/src/app/api/cron/cleanup/route.ts` — GET (UPLD-09, D-06):**

Add `export const runtime = 'nodejs';`
Add `export const dynamic = 'force-dynamic';` (prevent static caching of cron response).

This is NOT wrapped with `withApiKey` — it uses CRON_SECRET header auth instead.

Handler steps:
1. Validate `x-cron-secret` header: `if (req.headers.get('x-cron-secret') !== process.env.CRON_SECRET) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });` — also accept Vercel's `authorization: Bearer <CRON_SECRET>` header for Vercel Cron compatibility
2. `await connectDB()`
3. Find stale files: `const cutoff = new Date(Date.now() - 60 * 60 * 1000); const staleFiles = await File.find({ status: 'UPLOADING', createdAt: { $lt: cutoff } });`
4. Process each file with `Promise.allSettled`:
   - If `file.uploadId` is set (multipart in progress): call `AbortMultipartUploadCommand({ Bucket: R2_BUCKET, Key: file.key, UploadId: file.uploadId })` — catch and log errors (upload may already be aborted)
   - Then call `DeleteObjectCommand({ Bucket: R2_BUCKET, Key: file.key })` — catch and log errors (object may not exist if upload never completed)
   - Then `File.deleteOne({ _id: file._id })`
5. Log results: `console.log(\`[cleanup] Processed ${staleFiles.length} stale uploads\`)`
6. Return `NextResponse.json({ cleaned: staleFiles.length, timestamp: new Date().toISOString() })`

Import `DeleteObjectCommand`, `AbortMultipartUploadCommand` from `@aws-sdk/client-s3`.
Import `connectDB`, `File` from `@uploadkit/db`.

**3. Create or update `vercel.json` at project root:**
