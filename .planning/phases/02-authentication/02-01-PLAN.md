**Step 2: Create `packages/db/src/auth-client.ts`** — MongoClient bridge from Mongoose to @auth/mongodb-adapter.

Export an async function `getAuthMongoClient()` that:
1. Calls `connectDB()` from `./connection` to ensure the Mongoose connection is established
2. Returns `mongoose.connection.getClient()` cast as `MongoClient` from the `mongodb` package
3. The `mongodb` package is a transitive dependency of `@auth/mongodb-adapter` — do NOT add it to packages/db/package.json directly. Use `import type { MongoClient } from 'mongodb'` for the type.

```typescript
import mongoose from 'mongoose';
import { connectDB } from './connection';
import type { MongoClient } from 'mongodb';

export async function getAuthMongoClient(): Promise<MongoClient> {
  await connectDB();
  return mongoose.connection.getClient() as unknown as MongoClient;
}
```

import type { NextAuthConfig } from 'next-auth';
import GitHub from 'next-auth/providers/github';
import Google from 'next-auth/providers/google';
import Resend from 'next-auth/providers/resend';

export const authConfig = {
  pages: {
    signIn: '/login',
  },
  providers: [
    GitHub,

    },
  },

**Step 5: Create `apps/dashboard/auth.ts`** — Full config with adapter + database sessions.

This file runs in Node.js runtime only (route handlers, server components). It imports the adapter and Mongoose models.

Key implementation details per locked decisions:
- `adapter: MongoDBAdapter(await getAuthMongoClient())` — shares Mongoose connection pool (per D-04)
- `session.strategy: 'database'` with `maxAge: 30 * 24 * 60 * 60` (30 days per D-05)
- `session.updateAge: 24 * 60 * 60` — refresh session token once per day
- `events.createUser` callback auto-creates "My First Project" with a nanoid slug (per D-07). Include a guard: `const existing = await Project.findOne({ userId: user.id })` — skip creation if project exists (defense against duplicate events per Pitfall 4 from RESEARCH.md)
- `callbacks.session` must expose `user.id` on the session object (augment the session type)
- `callbacks.redirect` must send users to `/dashboard` after sign-in (per D-06). Only allow same-origin redirects (security: prevent open redirect).

Add NextAuth module augmentation at the top of auth.ts:
```typescript
declare module 'next-auth' {
  interface Session {
    user: {
      id: string;
    } & DefaultSession['user'];
  }
}
```

The full auth.ts:
```typescript
import NextAuth from 'next-auth';
import type { DefaultSession } from 'next-auth';
import { MongoDBAdapter } from '@auth/mongodb-adapter';
import { authConfig } from './auth.config';
import { getAuthMongoClient, connectDB, Project } from '@uploadkit/db';
import { nanoid } from 'nanoid';

declare module 'next-auth' {
  interface Session {
    user: {
      id: string;
    } & DefaultSession['user'];
  }
}

const authResult = NextAuth(async () => {
  const client = await getAuthMongoClient();
  return {
    ...authConfig,
    adapter: MongoDBAdapter(client),
    session: {
      strategy: 'database' as const,
      maxAge: 30 * 24 * 60 * 60,   // 30 days (D-05)
      updateAge: 24 * 60 * 60,     // refresh once per day
    },
    events: {
      async createUser({ user }) {
        // D-07: auto-create default project on first signup
        await connectDB();
        const existing = await Project.findOne({ userId: user.id });
        if (!existing) {
          await Project.create({
            name: 'My First Project',
            slug: `my-first-project-${nanoid(8)}`,
            userId: user.id,
          });
        }
      },
    },
    callbacks: {
      ...authConfig.callbacks,
      async session({ session, user }) {
        if (session.user) {
          session.user.id = user.id;
        }
        return session;
      },
      async redirect({ url, baseUrl }) {
        // D-06: after login, redirect to /dashboard
        // Security: only allow same-origin redirects
        if (url.startsWith(baseUrl)) return url;
        if (url.startsWith('/')) return `${baseUrl}${url}`;
        return `${baseUrl}/dashboard`;
      },
    },
  };
});

export const { handlers, auth, signIn, signOut } = authResult;
```

IMPORTANT: If the lazy config pattern `NextAuth(async () => {...})` is not supported by the current next-auth@beta version, fall back to:
```typescript
const client = await getAuthMongoClient();
export const { handlers, auth, signIn, signOut } = NextAuth({
  ...config
});
```
But this requires a top-level await. If neither works, create a `getAuth()` factory function and use it in the route handler. The RESEARCH.md documents this pattern as an alternative.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> proxy.ts | Unauthenticated requests hit Edge proxy — must check session cookie |
| Browser -> /api/auth/* | OAuth callbacks and magic link verification — Auth.js handles CSRF |
| Auth.js -> MongoDB | Session and user data stored — adapter manages collections |
| OAuth provider -> callback URL | External provider sends auth code — must validate state parameter |
| Email -> magic link URL | Verification token in URL — must be single-use, time-limited |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | OAuth callback | mitigate | Auth.js v5 validates state parameter and PKCE. Do not hand-roll OAuth flow. |
| T-02-02 | Spoofing | CSRF on sign-in form | mitigate | Auth.js v5 uses React Server Actions with built-in CSRF protection from Next.js. |
| T-02-03 | Tampering | Open redirect after OAuth | mitigate | redirect callback validates same-origin only: `if (url.startsWith(baseUrl)) return url; return baseUrl + '/dashboard'` |
| T-02-04 | Repudiation | Magic link replay | mitigate | Auth.js single-use VerificationToken — deleted after first use, expires in 24h. |
| T-02-05 | Information Disclosure | Session cookie leakage to subdomains | mitigate | Set AUTH_URL=https://dashboard.uploadkit.dev — cookies scoped to dashboard subdomain only. |
| T-02-06 | Elevation of Privilege | Session fixation | mitigate | Auth.js rotates session tokens on each sign-in. Old sessions invalidated. |
| T-02-07 | Denial of Service | Brute-force magic link requests | accept | Low risk — Resend has its own rate limits. Upstash rate limiter from Phase 1 can be wired to auth routes in Phase 3 if needed. |
| T-02-08 | Information Disclosure | AUTH_SECRET not set in production | mitigate | Document in .env.example with generation command. Auth.js warns in logs if missing. |
</threat_model>
